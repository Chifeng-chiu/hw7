全方位 JavaScript 實作挑戰：從基礎到後端邏輯

1
物件屬性存取 (Object Property Access)
目標：理解 post.title 的運作。
題目：宣告一個名為 post 的物件，包含 id: 1、title: "Hello World" 和 content: "Markdown content"。請練習用兩種方式印出 title：點符號 (Dot notation) 與中括號 (Bracket notation)。
const post = { id: 1, title: "Hello World", content: "Markdown content" }; console.log("Dot notation:", post.title); console.log("Bracket notation:", post["title"]);

解析

• Dot notation: post.title - 最直觀的寫法

• Bracket notation: post["title"] - 可使用變數動態存取

2
物件解構賦值 (Object Destructuring)
目標：理解程式中 const { title, content } = req.body; 的寫法。
題目：假設有一個變數 req 內容如下：
const req = { body: { title: "JS教學", content: "內容在此", author: "Gemini" } };

請用一行程式碼從 req.body 中取出 title 和 content 並宣告為同名常數。
const req = { body: { title: "JS教學", content: "內容在此", author: "Gemini" } }; const { title, content } = req.body; console.log(title, content); // JS教學 內容在此

解析

解構賦值可以快速從物件中取出多個屬性並宣告為變數。{ title, content } 等同於 { title: title, content: content }。

3
陣列的遍歷與字串拼接 (Array forEach & Template Literals)
目標：理解部落格首頁如何產生文章列表。
題目：給定一個陣列 const posts = [{id: 1, t: "A"}, {id: 2, t: "B"}]。請宣告一個空字串 let html = ""，並使用 forEach 遍歷陣列，將每個物件轉為 <div>A</div> 的格式拼接到 html 中。
const posts = [{id: 1, t: "A"}, {id: 2, t: "B"}]; let html = ""; posts.forEach(p => { html += `<div>${p.t}</div>`; }); console.log(html); // <div>A</div><div>B</div>

解析

使用模板字串（反引號）可以方便地嵌入變數。${p.t} 會動態替換成實際值。

4
字典與動態參數 (URL Params / Dictionary)
目標：理解 req.params.id 的來源。
題目：建立一個名為 params 的物件（字典），模擬 URL 參數。請動態新增一個鍵為 "id"，值為 99 的屬性，然後印出這個物件。
const params = {}; params["id"] = 99; console.log(params); // { id: 99 }

解析

JavaScript 物件本質上就是字典（key-value 儲存）。可以使用 物件["鍵"] = 值 動態新增屬性。

5
Callback 函數傳參 (Error-First Callback)
目標：理解 getPost(id, callback) 的非同步設計。
題目：撰寫一個函數 fetchData(id, callback)。
  • 邏輯：在函數內宣告一個物件 { id: id, status: "success" }。
  • 執行：呼叫 callback 並將 null (代表錯誤) 與該物件傳進去。
function fetchData(id, callback) { const fakeData = { id: id, status: "success" }; callback(null, fakeData); } fetchData(101, (err, data) => { if (err) console.log("發生錯誤：" + err); else console.log("成功取得資料：", data); });
提示：依照 Node.js 慣例，Callback 的第一個參數是用來放「錯誤訊息」的。如果沒有錯誤，就放 null。

解析

這是 Node.js 的「錯誤優先回呼」(Error-First Callback) 模式：
1. 第一個參數：錯誤（沒錯就傳 null）
2. 第二個參數：實際資料

6
JSON 處理 (Parsing JSON)
目標：理解 app.use(express.json()) 在處理什麼。
題目：給定一個 JSON 字串 const jsonStr = '{"title": "Post 1", "tags": ["js", "node"]}'。請將它轉換成 JavaScript 物件，並印出 tags 陣列中的第二個元素。
const jsonStr = '{"title": "Post 1", "tags": ["js", "node"]}'; const obj = JSON.parse(jsonStr); console.log(obj.tags[1]); // node

解析

JSON.parse() 將 JSON 字串轉換為 JavaScript 物件。express.json() 底層就是做這個轉換，讓我們能用 req.body 取得解析後的物件。

7
模擬資料庫查詢 (Simulating DB Queries)
目標：理解 db.get 的結構。
題目：寫一個函數 fakeGet(sql, params, callback)。
  • 內部邏輯：不管 SQL 是什麼，直接呼叫 callback(null, { title: "Fake Title" })。
  • 測試：呼叫這個函數，並在回呼函數中印出標題。
function fakeGet(sql, params, callback) { callback(null, { id: 1, title: "掌握 JavaScript 函數", content: "..." }); } const query = "SELECT * FROM posts WHERE id = ?"; const inputParams = [1]; fakeGet(query, inputParams, (err, row) => { if (err) console.error("查詢失敗"); else console.log("抓到的文章標題是：", row.title); });

解析

這模擬了 SQLite3 的 db.get(sql, params, callback) API。Callback 格式：(err, row) => { ... }

8
樣板字串中的邏輯運算 (Template Literals with Logic)
目標：理解網頁 HTML 模板的產生。
題目：宣告變數 user = "Guest"。請使用「反引號 (`)」建立一個 HTML 字串，內容為 <h1>Welcome, ${...}</h1>，其中 ${} 內要判斷：如果 user 有值就顯示 user，否則顯示 "Stranger"。
const user = "Guest"; const html = `<h1>Welcome, ${user || "Stranger"}</h1>`; console.log(html); // <h1>Welcome, Guest</h1>

解析

使用 || 運算子在模板字串中做三元運算：user 有值就用 user，否則用 "Stranger"。

9
陣列物件的排序與切片 (Sort & Substring)
目標：理解 SQL 語法在 JS 端的預習邏輯（例如 ORDER BY, substr）。
題目：給定一個陣列 ["Very long content here", "Another Very long content here", "3rd Very long content here"]。請取出這個字串的前 10 個字元，並在後方加上 "..."。
const arr = ["Very long content here", "Another Very long content here", "3rd Very long content here"]; const result = arr.map(s => s.substring(0, 10) + "..."); console.log(result); // ["Very long...", "Another Ve...", "3rd Very l..."]

解析

substring(0, 10) 取出字串的 0 到 9 位置（共 10 個字元）。

10
錯誤優先回呼模式 (Error-First Callback Pattern)
目標：理解程式中不斷出現的 if (err) return ...。
題目：寫一個函數 checkAdmin(role, callback)。
  • 如果 role 不是 "admin"，呼叫 callback("Access Denied")。
  • 如果是 "admin"，呼叫 callback(null, "Welcome")。

測試：練習呼叫此函數並處理有錯誤與沒錯誤的兩種狀況。

function checkAdmin(role, callback) { if (role !== "admin") callback("Access Denied"); else callback(null, "Welcome"); } checkAdmin("user", (err, msg) => { console.log(err || msg); }); checkAdmin("admin", (err, msg) => { console.log(err || msg); }); // Output: Access Denied, Welcome

解析

錯誤優先模式：先檢查 err，有錯誤就處理錯誤，沒錯誤才處理資料。
